# Midnight Signal APP

目前版本：2.4.4

適用於 SillyTavern 的獨立互動式手機介面。面板浮在聊天畫面上方，不會把 HTML 或按鈕內容塞進對話訊息。

## 功能

- 主頁直接顯示 SillyTavern 已匯入的角色卡，點擊即可查看角色主頁、完整設定與開場白。
- 讀取角色卡的預設開場白與 `alternate_greetings`，並直接套用到目前聊天的第一則角色訊息。
- 原「繼續對話」位置會直接顯示目前聊天與本次 API 呼叫的 Token 數字。
- 點擊 Token 按鈕可查看本次輸入、回覆、合計、目前聊天累計、全部累計與 API 回傳狀態。
- 對話：選擇角色後先顯示該角色的可捲動聊天室清單，點選其中一個才進入聊天畫面；可新增或重新命名每個聊天室。
- 訊息：可瀏覽目前聊天室內容、直接輸入及傳送訊息，並自行上傳聊天室背景。
- 聊天輸入區上方可切換目前 Chat Completion 來源的 AI 模型與補全預設設定檔。
- 關係：為每個角色保存獨立的關係備忘錄。
- 回憶：為每個角色新增、刪除獨立回憶。
- 動態：將近期角色回覆整理成動態卡片。
- 收藏：收藏常用角色。
- 設定：啟動時自動開啟、緊湊模式、聊天室字體、聊天室背景與清除 APP 筆記。
- 桌面版與手機版自適應，畫面中下方固定顯示「手機」啟動按鈕。

## 安裝

1. 解壓縮下載的 ZIP。
2. 將完整的 `midnight-signal-app` 資料夾放到：
   - 一般安裝：`SillyTavern/data/<你的使用者名稱>/extensions/`
   - 舊版單使用者安裝可能使用：`SillyTavern/public/scripts/extensions/third-party/`
3. 最終路徑必須是 `midnight-signal-app/manifest.json`，不可多包一層同名資料夾。
4. 重新啟動 SillyTavern，瀏覽器再按 `Ctrl + F5` 強制重新整理。
5. 在「擴充功能 → 管理擴充功能」確認 **Midnight Signal APP** 已啟用。
6. 點擊畫面中下方的「手機」按鈕開啟介面。

## 開場白切換說明

開場白資料直接讀取目前角色卡。角色卡只有一個開場白時，選單只會顯示「預設開場白」；若要更多選項，請在角色卡的進階定義中新增 Alternative Greetings。

切換開場白會修改目前聊天的第一則角色訊息。若你不想改動已進行中的劇情，建議先建立新聊天再選擇開場白。

## 相容性

- 針對 SillyTavern 1.18.0 製作。
- 最低客戶端版本設為 1.13.0。
- 不需要 Extras、伺服器外掛或外部 CDN。
- 角色切換優先使用 SillyTavern Context API，並保留核心模組與 DOM 後援方式。

- 點擊聊天室列後才會載入實際聊天畫面；聊天畫面的返回按鈕會回到該角色的聊天室清單。
- 可在清單中重新命名任一聊天室，也可在目前聊天室標題列直接改名。
- 可從聊天室清單新增空白聊天室；新聊天室的關係狀態、重要記憶與 Token 紀錄會建立為獨立資料。
- 重新命名只變更聊天室名稱，原有訊息、關係狀態、重要記憶與 Token 紀錄均完整保留。

## 疑難排解

- 看不到「手機」按鈕：確認資料夾層級、擴充已啟用，並按 `Ctrl + F5`。
- 角色清單是空的：請先在 SillyTavern 匯入至少一張角色卡。
- 不能切換角色：請先離開群組聊天；這個介面目前只切換單人角色。
- 開場白無法套用：確認目前聊天的第一則訊息是角色開場白，而不是玩家訊息。
