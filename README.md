# Midnight Signal APP

目前版本：2.0.0

適用於 SillyTavern 的獨立互動式手機介面。面板浮在聊天畫面上方，不會把 HTML 或按鈕內容塞進對話訊息。

## 功能

- 主頁直接顯示 SillyTavern 已匯入的角色卡，點擊即可查看角色主頁、完整設定與開場白。
- 讀取角色卡的預設開場白與 `alternate_greetings`，並直接套用到目前聊天的第一則角色訊息。
- 原「繼續對話」位置會直接顯示目前聊天與本次 API 呼叫的 Token 數字。
- 點擊 Token 按鈕可查看本次輸入、回覆、合計、目前聊天累計、全部累計與 API 回傳狀態。
- 訊息：可瀏覽目前聊天內容、直接輸入及傳送訊息，並自行上傳聊天室背景。
- 聊天輸入區上方可切換目前 Chat Completion 來源的 AI 模型與補全預設設定檔。
- 關係：為每個角色保存獨立的關係備忘錄。
- 回憶：為每個角色新增、刪除獨立回憶。
- 動態：將近期角色回覆整理成動態卡片。
- 收藏：收藏常用角色。
- 設定：啟動時自動開啟、緊湊模式、聊天室字體、聊天室背景與清除 APP 筆記。
- 桌面版與手機版自適應，畫面中下方固定顯示「手機」啟動按鈕。

## 安裝

1. 解壓縮下載的 ZIP。
2. 將完整的 `midnight-signal-app` 資料夾放到：
   - 一般安裝：`SillyTavern/data/<你的使用者名稱>/extensions/`
   - 舊版單使用者安裝可能使用：`SillyTavern/public/scripts/extensions/third-party/`
3. 最終路徑必須是 `midnight-signal-app/manifest.json`，不可多包一層同名資料夾。
4. 重新啟動 SillyTavern，瀏覽器再按 `Ctrl + F5` 強制重新整理。
5. 在「擴充功能 → 管理擴充功能」確認 **Midnight Signal APP** 已啟用。
6. 點擊畫面中下方的「手機」按鈕開啟介面。

## 開場白切換說明

開場白資料直接讀取目前角色卡。角色卡只有一個開場白時，選單只會顯示「預設開場白」；若要更多選項，請在角色卡的進階定義中新增 Alternative Greetings。

切換開場白會修改目前聊天的第一則角色訊息。若你不想改動已進行中的劇情，建議先建立新聊天再選擇開場白。

## 相容性

- 針對 SillyTavern 1.18.0 製作。
- 最低客戶端版本設為 1.13.0。
- 不需要 Extras、伺服器外掛或外部 CDN。
- 角色切換優先使用 SillyTavern Context API，並保留核心模組與 DOM 後援方式。

## 1.0.1 修正

- 最近訊息頁不再截斷訊息內容，完整保留全部文字與換行。
- 主視覺圖片改由擴充樣式表的相對路徑載入，修正不同使用者資料夾或掛載路徑下圖片無法顯示的問題。

## 1.0.2 雲端版修正

- 主視覺已直接嵌入樣式表，不再讀取外部 PNG 檔案或靜態素材網址。
- 適用於只允許載入擴充 JavaScript／CSS、無法公開附加圖片路徑的雲端環境。

## 1.1.0 Token 統計

- 將原「繼續對話」按鈕改為 Token 數字顯示按鈕。
- 整合 API 回傳的 usage 資料，支援一般 JSON 與 SSE 串流回應。
- 詳細頁可查看輸入、回覆、本次合計、聊天累計、全部累計、API 呼叫次數與玩家訊息數。
- 若已安裝獨立的「API Token 用量面板」，會共用既有統計資料並避免再次攔截 API 請求。

## 1.2.0 APP 內對話

- 「訊息」頁新增聊天輸入框與傳送按鈕，可直接在 Midnight Signal APP 內進行對話。
- 訊息仍交由 SillyTavern 原生傳送流程處理，會套用目前角色、世界書、提示詞、模型與 Token 統計。
- 支援 Enter 傳送、Shift+Enter 換行；新訊息與角色回覆會自動更新並捲至最新內容。
- 為維持長對話效能，APP 顯示最近 100 則訊息，更早內容仍完整保存在 SillyTavern 聊天紀錄。

## 1.3.0 雲端傳送與 Token 計算修正

- 首頁原「繼續對話」位置固定顯示 **TOKEN USAGE**，並列出目前聊天／本次 API Token 數字。
- APP 內傳送改為直接呼叫 SillyTavern 核心 `Generate('normal')`，不再依賴模擬點擊雲端版可能不存在的傳送按鈕。
- Token 詳細頁保留 API usage 精確數字，並新增 SillyTavern 當前 tokenizer 計算的「目前聊天內容 Token」。
- 若供應商沒有回傳 usage，仍可查看聊天內容的 tokenizer 計算值，並會清楚標示資料來源。
- manifest 已綁定 `finna-cloud/20260711-1` 並啟用擴充自動更新。

## 1.3.2 APP 內人設管理

- 編輯結果會同步回 SillyTavern 原生人設資料與目前提示詞，不會只停留在 APP 畫面。

## 1.3.3 聊天室字體大小

## 1.3.4 角色卡管理

- 底部導覽新增「角色」頁面，可查看並切換目前所有角色卡。
- 可直接呼叫 SillyTavern 原生匯入器，支援 PNG、JSON、YAML、YML、CHARX 與 BYAF。
- 可在 APP 內填寫名稱、頭像、描述、性格、場景與開場白並建立新角色卡。
- 角色清單提供刪除按鈕；刪除前會再次確認，並預設保留既有聊天紀錄。

## 1.4.0 角色主頁與聊天外觀／模型切換

- 主頁新增可橫向滑動的角色卡區塊，點擊角色卡可進入角色專屬主頁，直接查看描述、性格、場景與預設開場白。

- 模型、預設與角色選單皆使用適合手機觸控的底部面板，長選單可獨立捲動。

## 疑難排解

- 看不到「手機」按鈕：確認資料夾層級、擴充已啟用，並按 `Ctrl + F5`。
- 角色清單是空的：請先在 SillyTavern 匯入至少一張角色卡。
- 不能切換角色：請先離開群組聊天；這個介面目前只切換單人角色。
- 開場白無法套用：確認目前聊天的第一則訊息是角色開場白，而不是玩家訊息。
